# AI Assistant для Системы Электронного Документооборота (СЭД)

## 📋 О проекте

Интеллектуальный ассистент на основе больших языковых моделей (LLM) для поддержки пользователей систем электронного документооборота (СЭД). Проект является основой для диссертационного исследования на тему: **"Разработка интеллектуального ассистента для поддержки пользователей систем электронного документооборота на основе LLM"**.

### Целевая аудитория
- Сотрудники организаций, работающие с СЭД
- Новые пользователи, проходящие обучение
- Специалисты, нуждающиеся в быстром доступе к регламентам

### Основная идея
Ассистент предоставляет **контекстно-ориентированную поддержку**, объединяя:
- Текущее состояние документа и бизнес-процесса
- Роль пользователя в системе
- Релевантную информацию из регламентов и инструкций (RAG)

Ассистент **не выполняет действий** в СЭД, а лишь консультирует пользователя на естественном языке.

## 🎯 Смысл разработки

### Научная новизна
Предложен **метод контекстно-ориентированной поддержки** пользователей СЭД на основе LLM, который:
1. **Расширяет пользовательский запрос** контекстом из системы:
   - Данные документа (тип, статус, этап, ответственные)
   - Состояние бизнес-процесса
   - Релевантные фрагменты из регламентирующей документации (через семантический поиск)

2. **Формирует augmented prompt**, объединяющий все источники контекста для генерации персонализированного ответа

### Практическая ценность
- **Ускорение обучения** новых сотрудников
- **Снижение количества ошибок** при работе с документами
- **Быстрый доступ к знаниям** без поиска в множестве документов
- **Понимание контекста** — не шаблонные ответы, а учёт текущей ситуации

### Отличие от существующих решений
- **Не правила и скрипты**, а естественный язык
- **Не изолированные ответы**, а учёт контекста документа и пользователя
- **Не поиск по ключевым словам**, а семантическое понимание запроса

## 🔄 Pipeline обработки запроса

Каждый пользовательский запрос проходит через 6 шагов:

### 1. **ClassificationStep**
Определяет тип запроса:
- `InstructionQuery` — "Как создать доверенность?"
- `BusinessProcessQuery` — "Почему документ вернулся?"
- `DocumentSearchQuery` — "Найди договоры от января"
- `ErrorAnalysisQuery` — "Почему не могу подписать?"

### 2. **ContextExtractionStep**
Извлекает из mock-СЭД:
- **DocumentContext**: тип, статус, этап, автор, ответственный
- **UserContext**: роль, подразделение, права доступа
- **BusinessProcess**: текущая стадия процесса

### 3. **RagSearchStep**
Семантический поиск в Qdrant:
- Векторизует запрос через Ollama (nomic-embed-text)
- Находит релевантные фрагменты документации
- Возвращает топ-5 результатов с метаданными

### 4. **PromptBuildStep** ⭐ (научная новизна)
Формирует **расширенный промпт**, объединяя:
```
Системная роль: ты ассистент СЭД

Контекст документа:
- Тип: доверенность
- Статус: возврат
- Этап: проверка паспортных данных
- Ответственный: отдел кадров

Информация из регламентов:
1. При отсутствии паспортных данных доверенность возвращается
2. Документ должен содержать серию и номер паспорта

Вопрос пользователя:
Почему мой документ вернулся?
```

### 5. **LlmGenerationStep**
Отправляет промпт в Ollama (qwen2.5:7b) и получает ответ

### 6. **ExperimentLoggingStep**
Сохраняет метрики для A/B сравнения (с контекстом vs без)

### Запуск системы

1. **Клонировать репозиторий**:
```bash
git clone https://github.com/keyloud/AI-Assistant-SED.git
cd AI-Assistant-SED
```

2. **Настроить переменные окружения** (опционально):
```bash
cp .env.example .env
# Отредактировать .env при необходимости
```

3. **Запустить все сервисы**:
```bash
docker-compose up -d
```

4. **Дождаться готовности сервисов**:
```bash
# Проверить статус
docker-compose ps

# Проверить логи
docker-compose logs -f assistant-api
```

5. **Загрузить модели в Ollama** (первый запуск):
```bash
docker exec -it ollama ollama pull qwen2.5:7b
docker exec -it ollama ollama pull nomic-embed-text
```

6. **Индексировать базу знаний**:
```bash
cd knowledge-base/ingestion
python ingest.py
```

7. **Открыть UI**:
```
http://localhost:5173
```

## 🧪 4 сценария использования

### 1. Ответы по инструкциям
**Вопрос**: "Как создать доверенность?"
**Ассистент**:
- Классифицирует как `InstructionQuery`
- Ищет в регламентах инструкции по созданию доверенностей
- Возвращает пошаговое руководство

### 2. Помощь по бизнес-процессу
**Вопрос**: "Почему документ вернулся?"
**Ассистент**:
- Классифицирует как `BusinessProcessQuery`
- Извлекает контекст документа (статус: "возврат", этап: "проверка паспортных данных")
- Ищет в регламентах причины возврата на этом этапе
- Объясняет причину с учётом контекста

### 3. Поиск документов
**Вопрос**: "Найди договоры от января"
**Ассистент**:
- Классифицирует как `DocumentSearchQuery`
- Выполняет поиск в mock-СЭД по фильтрам
- Структурирует и возвращает список документов

### 4. Анализ ошибок
**Вопрос**: "Почему не могу подписать документ?"
**Ассистент**:
- Классифицирует как `ErrorAnalysisQuery`
- Проверяет права доступа пользователя
- Проверяет состояние бизнес-процесса
- Объясняет причину (например, "документ уже подписан" или "нет прав")

## 📊 Эксперименты и оценка (Фаза 6)

### Методология
A/B сравнение двух вариантов:
- **Вариант A**: ответы **без контекста** (baseline)
- **Вариант B**: ответы **с контекстом** (предлагаемый метод)

### Метрики
- Точность ответов (ручная оценка)
- Релевантность (сравнение с эталоном)
- Время отклика
- Полнота использования контекста
