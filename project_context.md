# Контекст проекта: AI ассистент для СЭД

## 1. Тема диссертации
"Разработка интеллектуального ассистента для поддержки пользователей систем электронного документооборота на основе LLM"

## 2. Научная новизна
Предложен метод контекстно-ориентированной поддержки пользователей СЭД на основе LLM, учитывающий:
- роль пользователя
- состояние бизнес-процесса документа
- семантический поиск по регламентирующей документации

Метод основан на **расширении пользовательского запроса контекстом системы**, включая данные документа, состояние бизнес-процесса и результаты поиска по нормативной документации.

## 3. Цель MVP
- Поддержка пользователей через **чат**.
- Ассистент знает контекст документа, анализирует его и подсказывает.
- Основные сценарии:
  1. Ответы по инструкциям
  2. Помощь по бизнес-процессу документа
  3. Поиск документов
  4. Анализ ошибок пользователя
- Исключено выполнение действий.

## 4. Данные и контент
- Собственные инструкции и регламенты (5–10 документов для теста)
- Описание бизнес-процессов
- Тестовые документы: сгенерировать или взять с Kaggle
- Контекст документа пример:

```json
{
  "documentType": "доверенность",
  "status": "возврат",
  "stage": "проверка паспортных данных",
  "author": "Иванов",
  "responsible": "Отдел кадров"
}

## 5.Архитектура системы
- UI: отдельный веб-сервис, чат справа (Streamlit)
- Backend: C# ASP.NET Core API
- LLM: локальная модель (Ollama / Qwen 2.5 7B)
- Vector DB: Qdrant для RAG
- СЭД: Docsvision (игровой стенд) или абстрактная модель документов

Схема:

Chat UI (Streamlit)
       |
Assistant API (C# ASP.NET)
       |
--------------------------
|           |            |
LLM       Vector DB      SED API

6. Алгоритм обработки запроса
- Получение запроса пользователя
- Определение типа запроса (инструкция, документ, поиск, ошибка)
- Извлечение контекста документа и пользователя из СЭД
- Семантический поиск по базе знаний (RAG)
- Формирование расширенного промпта с контекстом и найденными документами
- Генерация ответа LLM
- Возврат ответа пользователю

Пример промпта
Ты ассистент системы электронного документооборота.

Контекст документа:
Тип: доверенность
Статус: возврат
Этап: проверка паспортных данных
Ответственный: отдел кадров

Информация из регламентов:
1. При отсутствии паспортных данных доверенность возвращается.
2. Документ должен содержать серию и номер паспорта.

Вопрос пользователя:
Почему мой документ вернулся?

Ответь кратко и понятно.
